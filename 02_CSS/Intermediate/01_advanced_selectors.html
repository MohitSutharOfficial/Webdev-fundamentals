🎯 Advanced CSS Selectors Mastery

📚 Beyond Basic Selectors

Advanced CSS selectors give you precise control over element targeting. They enable complex styling patterns and dynamic interactions without JavaScript.

🎯 What You'll Master:

  • Structural Pseudo-classes - Target elements by position
  • Advanced Attribute Selectors - Complex attribute matching
  • Negation & Functional Selectors - :not(), :is(), :where()
  • Form State Selectors - Dynamic form styling
  • Complex Combinators - Multi-level element relationships

🔢 Structural Pseudo-classes

nth-child() Selectors

/* Select specific positions */ li:nth-child(3) { color: red; } /* 3rd child */ li:nth-child(odd) { background: #f0f0f0; } /* odd children */ li:nth-child(even) { background: white; } /* even children */ /* Mathematical expressions */ li:nth-child(3n) { color: blue; } /* every 3rd: 3, 6, 9, 12... */ li:nth-child(3n+1) { color: red; } /* 1, 4, 7, 10, 13... */ li:nth-child(3n+2) { color: green; } /* 2, 5, 8, 11, 14... */ /* From the end */ li:nth-last-child(2) { font-weight: bold; } /* 2nd from end */

nth-of-type() Selectors

/* Select by element type, not position */ h2:nth-of-type(odd) { color: red; } /* odd h2 elements */ p:nth-of-type(3) { font-style: italic; } /* 3rd p element */ img:nth-of-type(2n) { border: 2px solid blue; } /* even images */

🏷️ Advanced Attribute Selectors

/* Exact match */ [data-status="active"] { background: green; } /* Starts with */ [data-priority^="high"] { border-left: 5px solid red; } /* Ends with */ [data-category$="end"] { border-right: 5px solid blue; } /* Contains substring */ [data-tags*="important"] { font-weight: bold; } /* Word match (space-separated) */ [class~="special"] { color: purple; } /* Language attribute match */ [lang|="en"] { font-style: italic; } /* Case-insensitive match */ [data-type="button" i] { cursor: pointer; }

🚫 Negation Pseudo-class

/* Select all paragraphs except those with class 'special' */ p:not(.special) { color: gray; font-style: italic; } /* Select all inputs except checkboxes */ input:not([type="checkbox"]) { border: 2px solid blue; } /* Select all elements except the first child */ .container > *:not(:first-child) { margin-top: 20px; } /* Multiple negations */ button:not(.primary):not(.secondary) { background: #ccc; }

📝 Form State Selectors

/* Form validation states */ input:valid { border: 2px solid green; } input:invalid { border: 2px solid red; } /* User interaction states */ input:focus { box-shadow: 0 0 0 3px rgba(0,123,255,.25); } input:disabled { opacity: 0.5; cursor: not-allowed; } /* Checkbox and radio states */ input:checked + label { background: blue; color: white; } input:indeterminate { background: orange; } /* Form requirement states */ input:required { border-left: 3px solid red; } input:optional { border-left: 3px solid green; } /* Range and number inputs */ input:in-range { background: lightgreen; } input:out-of-range { background: lightcoral; }

🎯 Target Pseudo-class

/* Style elements when they are the target of a URL fragment */ .section:target { background: yellow; border: 2px solid orange; padding: 20px; } /* Useful for creating tabs or accordion without JavaScript */ .tab-content:target { display: block; } .tab-content:not(:target) { display: none; }

🔗 Complex Combinators

/* Multiple combinator types */ .header nav ul li a:hover { color: blue; } /* Adjacent sibling after direct child */ .parent > .child + .sibling { margin-top: 0; } /* General sibling with class */ .trigger ~ .target.active { display: block; } /* Complex attribute and pseudo-class combination */ input[type="text"]:focus:valid + .success-message { display: block; color: green; }

🎨 Modern Selector Functions

/* :is() - Matches any of the selectors in the list */ :is(h1, h2, h3, h4, h5, h6) { line-height: 1.2; margin-bottom: 1em; } /* :where() - Same as :is() but with zero specificity */ :where(h1, h2, h3, h4, h5, h6) { color: #333; } /* :has() - Parent selector (future CSS) */ .card:has(img) { border: 2px solid blue; } /* :not() with multiple selectors */ button:not(.primary, .secondary) { background: #ccc; }

🎯 Browser Support Note:

Some modern selectors like :has() have limited browser support. Always check compatibility on Can I Use.

📊 Advanced Selector Reference

Selector Description Example
:nth-child(n) Selects every nth child element li:nth-child(3)
:nth-of-type(n) Selects every nth element of its type p:nth-of-type(2)
:not(selector) Selects elements that don't match div:not(.special)
:target Selects the targeted element section:target
[attr^="value"] Attribute starts with value [href^="https"]
[attr$="value"] Attribute ends with value [href$=".pdf"]
[attr*="value"] Attribute contains value [class*="btn"]
:is(selector) Matches any selector in the list :is(h1, h2, h3)

🎯 Best Practices

✅ DO:

  • Use semantic data attributes for JavaScript hooks
  • Prefer :nth-child() over complex class naming
  • Use :not() to reduce CSS duplication
  • Test advanced selectors across browsers
  • Document complex selector patterns
  • Use CSS custom properties with attribute selectors

❌ DON'T:

  • Over-complicate selectors unnecessarily
  • Rely on browser-specific selectors
  • Use :nth-child() when simple classes work better
  • Forget about performance with complex selectors
  • Mix too many pseudo-classes in one selector
  • Ignore fallbacks for modern selectors

🚨 Performance Considerations

⚠️ Selector Performance Tips:

  • Right-to-left reading: Browsers read selectors from right to left
  • Avoid universal selectors: * { } can be slow
  • Limit nesting depth: Keep selectors shallow
  • Use IDs and classes: Faster than attribute selectors
  • Avoid complex :nth-child(): Simple numbers are faster
/* Slow */ div > div > div > p:nth-child(3n+1) { } /* Better */ .content-item:nth-child(3n+1) { } /* Slow */ *[data-*] { } /* Better */ .component[data-state="active"] { }
