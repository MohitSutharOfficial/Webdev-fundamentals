1. Repeat Function and Auto-fit/Auto-fill

/* Auto-responsive columns */ .auto-grid { display: grid; /* Creates as many columns as can fit, minimum 250px wide */ grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* Alternative with auto-fill (creates empty columns) */ /* grid-template-columns: repeat(auto-fill, minmax(250px, 1fr)); */ gap: 20px; } /* Advanced repeat patterns */ .pattern-grid { display: grid; grid-template-columns: repeat(3, 100px 200px); /* 100px 200px 100px 200px 100px 200px */ gap: 15px; } /* Fractional units with fixed sizes */ .mixed-grid { display: grid; grid-template-columns: 250px repeat(3, 1fr) 150px; gap: 20px; }

2. CSS Grid with Modern Functions

/* Modern CSS functions in Grid */ .modern-grid { display: grid; /* Clamp for responsive sizing */ grid-template-columns: repeat(auto-fit, minmax(clamp(200px, 30vw, 400px), 1fr)); /* Dynamic gap based on viewport */ gap: clamp(10px, 2vw, 30px); /* Min/max for track sizing */ grid-template-rows: minmax(100px, auto) 1fr minmax(50px, auto); } /* CSS custom properties for dynamic grids */ .custom-grid { --columns: 3; --gap: 20px; --min-width: 250px; display: grid; grid-template-columns: repeat(var(--columns), minmax(var(--min-width), 1fr)); gap: var(--gap); } /* Container query integration */ @container (min-width: 600px) { .custom-grid { --columns: 4; --gap: 25px; } }

/* Combining Grid and Flexbox */ .hybrid-container { display: grid; grid-template-columns: 250px 1fr; gap: 20px; } .sidebar { /* Grid item that uses flexbox internally */ display: flex; flex-direction: column; gap: 15px; } .main-content { /* Grid item with flex layout for cards */ display: flex; flex-direction: column; gap: 20px; } .card-container { /* Flexbox for card layout within grid */ display: flex; gap: 15px; flex-wrap: wrap; } .card { flex: 1 1 300px; /* grow, shrink, basis */ min-width: 0; /* Allow shrinking below content size */ }

/* Mobile-first responsive grid */ .responsive-layout { display: grid; gap: 1rem; /* Mobile: single column */ grid-template-columns: 1fr; grid-template-areas: "header" "main" "sidebar" "footer"; } /* Tablet: 2-column layout */ @media (min-width: 768px) { .responsive-layout { grid-template-columns: 200px 1fr; grid-template-areas: "header header" "sidebar main" "footer footer"; } } /* Desktop: 3-column layout */ @media (min-width: 1024px) { .responsive-layout { grid-template-columns: 200px 1fr 150px; grid-template-areas: "header header header" "sidebar main aside" "footer footer footer"; } } /* Container queries (future/experimental) */ @container (min-width: 600px) { .card-grid { grid-template-columns: repeat(2, 1fr); } } @container (min-width: 900px) { .card-grid { grid-template-columns: repeat(3, 1fr); } }

üöÄ Senior Engineer Tips:

  • Grid vs Flexbox: Use Grid for 2D layouts, Flexbox for 1D layouts and component-level design
  • Performance: Avoid complex calc() functions in grid definitions; prefer simpler fractional units
  • Accessibility: Maintain logical tab order even when visually reordering with grid
  • Browser Support: Use @supports for fallbacks when using cutting-edge features
  • Debugging: Use Firefox Grid Inspector for visualizing grid lines and areas
  • Naming: Use meaningful grid area names for better maintainability
  • Container Queries: Prepare for container queries by thinking component-first

‚ö†Ô∏è Common Grid Pitfalls:

  • Overusing grid for simple layouts where flexbox would suffice
  • Not considering accessibility when reordering content visually
  • Creating overly complex grid definitions that are hard to maintain
  • Forgetting to test responsive behavior across different screen sizes
  • Not providing fallbacks for older browsers when needed
  • Using CSS Grid for layouts that need to support IE11 without proper fallbacks
