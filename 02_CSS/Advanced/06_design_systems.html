Implementation Guide

Token Usage

/* Using design tokens in CSS */ .custom-component { padding: var(--space-4) var(--space-6); background-color: var(--color-primary-50); border: 1px solid var(--color-primary-200); border-radius: var(--radius-md); font-size: var(--font-size-base); color: var(--color-neutral-800); box-shadow: var(--shadow-sm); } .custom-component:hover { background-color: var(--color-primary-100); box-shadow: var(--shadow-md); transition: all var(--duration-200) var(--ease-in-out); }

Theme Implementation

/* Theme switching with CSS custom properties */ :root { --color-background: #ffffff; --color-text: #1f2937; } [data-theme="dark"] { --color-background: #1f2937; --color-text: #f9fafb; } /* Components automatically adapt to theme changes */ .component { background-color: var(--color-background); color: var(--color-text); }

JavaScript Integration

// Reading design tokens in JavaScript const primaryColor = getComputedStyle(document.documentElement) .getPropertyValue('--color-primary-600'); // Setting design tokens dynamically document.documentElement.style.setProperty( '--color-primary-600', '#3b82f6' ); // Theme switching function toggleTheme() { const currentTheme = document.documentElement.getAttribute('data-theme'); const newTheme = currentTheme === 'dark' ? 'light' : 'dark'; document.documentElement.setAttribute('data-theme', newTheme); }

Design System Benefits

🎨 Consistency

Design tokens ensure visual consistency across all components and platforms by centralizing design decisions.

🔧 Maintainability

Changes to design tokens automatically propagate throughout the system, making updates efficient and error-free.

⚡ Scalability

Component-based architecture with design tokens scales efficiently as teams and products grow.

🤝 Collaboration

Shared design language improves communication between designers, developers, and stakeholders.
