1. Details and Summary

<!-- Native collapsible content --> <details> <summary>Click to expand</summary> <p>This content is hidden by default and can be toggled open/closed.</p> <ul> <li>No JavaScript required</li> <li>Semantic and accessible</li> <li>Styleable with CSS</li> </ul> </details> <!-- Open by default --> <details open> <summary>Expanded by default</summary> <p>This starts in the open state.</p> </details>

What is semantic HTML?

Semantic HTML refers to using HTML markup to reinforce the semantics, or meaning, of the information in webpages and web applications rather than merely to define its presentation or look.

Why use modern HTML features?

Modern HTML features provide:

  • Better accessibility out of the box
  • Reduced need for JavaScript
  • Improved performance
  • Better SEO and semantic meaning
Browser support considerations

Always check browser support for new features and provide fallbacks when necessary. Use progressive enhancement to ensure your site works for all users.

📱 Web Components (Custom Elements)

<!-- Define a custom element --> <script> class MyCustomElement extends HTMLElement { constructor() { super(); this.attachShadow({ mode: 'open' }); this.shadowRoot.innerHTML = ` <style> :host { display: block; padding: 20px; border: 2px solid #3498db; border-radius: 5px; background: linear-gradient(45deg, #f8f9fa, #e9ecef); } .content { font-family: Arial, sans-serif; } .title { color: #2c3e50; margin: 0 0 10px 0; } </style> <div class="content"> <h3 class="title">${this.getAttribute('title') || 'Custom Element'}</h3> <slot></slot> </div> `; } static get observedAttributes() { return ['title']; } attributeChangedCallback(name, oldValue, newValue) { if (name === 'title') { const titleElement = this.shadowRoot.querySelector('.title'); if (titleElement) { titleElement.textContent = newValue || 'Custom Element'; } } } } customElements.define('my-custom-element', MyCustomElement); </script> <!-- Use the custom element --> <my-custom-element title="Hello Web Components"> <p>This content is projected into the slot.</p> <button>Custom Button</button> </my-custom-element>

🌐 Browser Support Notes:

  • Dialog Element: Modern browsers (Chrome 37+, Firefox 98+, Safari 15.4+)
  • Details/Summary: Excellent support across all modern browsers
  • Lazy Loading: Chrome 76+, Firefox 75+, Safari 15.4+
  • Custom Elements: Good support in modern browsers with polyfills available
  • Input Types: Graceful degradation to text inputs in older browsers

🚀 Senior Engineer Tips:

  • Progressive Enhancement: Use modern features as enhancements, not dependencies
  • Feature Detection: Use `if ('showModal' in document.createElement('dialog'))` to detect support
  • Polyfills: Provide fallbacks for critical functionality in older browsers
  • Accessibility: Modern elements often have better built-in accessibility
  • Performance: Native implementations are typically faster than JavaScript alternatives
  • Web Standards: Stay updated with new specifications and browser implementations

⚠️ Implementation Considerations:

  • Always check browser support before using new features
  • Provide fallbacks for critical functionality
  • Test across different devices and browsers
  • Consider using progressive enhancement strategies
  • Be aware of differences in implementation across browsers
  • Keep accessibility in mind when using new features
