üì± Responsive Images

Modern websites must serve appropriate images for different screen sizes and resolutions.

Responsive Image Techniques:

1. Simple Responsive (CSS max-width):
<img src="large-image.jpg" alt="Descriptive text" style="max-width: 100%; height: auto;">
2. Srcset for Different Resolutions:
<img src="image-400w.jpg" srcset="image-400w.jpg 400w, image-800w.jpg 800w, image-1200w.jpg 1200w" sizes="(max-width: 400px) 100vw, (max-width: 800px) 80vw, 1200px" alt="Responsive image example">
3. Picture Element for Art Direction:
<picture> <!-- Mobile: Square crop --> <source media="(max-width: 600px)" srcset="mobile-square.jpg"> <!-- Tablet: 16:9 crop --> <source media="(max-width: 1024px)" srcset="tablet-wide.jpg"> <!-- Desktop: Full landscape --> <img src="desktop-full.jpg" alt="Landscape photo adapting to screen size"> </picture>
4. Modern Format Support:
<picture> <!-- WebP for modern browsers --> <source srcset="image.webp" type="image/webp"> <!-- AVIF for cutting-edge browsers --> <source srcset="image.avif" type="image/avif"> <!-- Fallback JPEG for older browsers --> <img src="image.jpg" alt="Cross-browser compatible image"> </picture>

üéØ Responsive Image Strategy:

  1. Plan your breakpoints - Design for mobile, tablet, desktop
  2. Create multiple sizes - Usually 3-5 different widths
  3. Optimize file sizes - Use tools like ImageOptim or Squoosh
  4. Test on real devices - Simulator isn't always accurate

‚ôø Image Accessibility

Making images accessible to all users, including those using screen readers.

Alt Text Best Practices:

Informative Image
Informative Image:
alt="Bar chart showing 40% increase in sales"

Describe the information the image conveys

Decorative Image
Decorative Image:
alt="" or role="presentation"

Empty alt for images that don't add information

Functional Image
Functional Image:
alt="Search"

Describe the function, not the image

Complex Images (Charts, Diagrams):

<figure role="img" aria-labelledby="chart-title" aria-describedby="chart-desc"> <img src="sales-chart.png" alt="Sales chart"> <figcaption id="chart-title"> Quarterly Sales Performance 2024 </figcaption> <p id="chart-desc"> Detailed description: Q1 sales were $100k, Q2 increased to $150k, Q3 reached $200k, and Q4 achieved $250k, showing consistent growth. </p> </figure>

‚ö†Ô∏è Common Alt Text Mistakes:

  • Redundant phrases: Don't use "image of" or "picture of"
  • File names: Don't use "IMG_1234.jpg" as alt text
  • Too long: Keep alt text under 125 characters when possible
  • Missing context: Alt text should fit the surrounding content

Lazy Loading and Performance:

<!-- Native lazy loading --> <img src="image.jpg" loading="lazy" decoding="async" alt="Loads when user scrolls near"> <!-- Intersection Observer for advanced lazy loading --> <img data-src="large-image.jpg" src="placeholder.jpg" alt="Advanced lazy loaded image" class="lazy-load"> <script> // Intersection Observer for lazy loading const imageObserver = new IntersectionObserver((entries, observer) => { entries.forEach(entry => { if (entry.isIntersecting) { const img = entry.target; img.src = img.dataset.src; img.classList.remove('lazy-load'); observer.unobserve(img); } }); }); document.querySelectorAll('.lazy-load').forEach(img => { imageObserver.observe(img); }); </script>

Progressive Enhancement:

<!-- Progressive JPEG with quality fallbacks --> <picture> <source srcset="image-high-quality.jpg" media="(min-width: 1200px)"> <source srcset="image-medium-quality.jpg" media="(min-width: 600px)"> <img src="image-low-quality.jpg" alt="Progressive quality image"> </picture> <!-- SVG with PNG fallback --> <object data="icon.svg" type="image/svg+xml"> <img src="icon.png" alt="Scalable icon with fallback"> </object>

‚ö° Image Optimization Strategies

Performance is crucial for modern web applications. Optimize your images for speed.

File Format Selection Guide:

  • JPEG: Photos with many colors, gradients
  • PNG: Images with transparency, simple graphics
  • WebP: Modern format, 25-50% smaller than JPEG/PNG
  • AVIF: Next-gen format, even smaller than WebP
  • SVG: Scalable graphics, icons, simple illustrations

Performance Optimization Checklist:

  • ‚úÖ Compress images (use tools like Squoosh, ImageOptim)
  • ‚úÖ Choose appropriate dimensions (don't scale large images with CSS)
  • ‚úÖ Use modern formats (WebP, AVIF) with fallbacks
  • ‚úÖ Implement lazy loading for below-fold images
  • ‚úÖ Specify width and height to prevent layout shift
  • ‚úÖ Use CDN for faster global delivery
  • ‚úÖ Consider critical images for preloading

üîß Troubleshooting Common Issues

Issue: Images Not Loading

Causes:

  • Incorrect file path (check relative vs absolute paths)
  • File doesn't exist or was moved
  • Server permissions issues
  • CORS restrictions for cross-origin images

Solutions:

  • Verify file paths in browser developer tools
  • Check network tab for 404 errors
  • Use relative paths when possible
  • Implement proper error handling with onerror attribute

Issue: Poor Performance

Causes:

  • Large file sizes
  • Too many HTTP requests
  • Unoptimized images
  • Missing lazy loading

Solutions:

  • Compress images before uploading
  • Use appropriate file formats
  • Implement lazy loading
  • Use CDN for faster delivery
