📦 Variable Declarations (var, let, const)

Variables are containers that store data. JavaScript has three ways to declare variables:

// Modern JavaScript - Use these! const PI = 3.14159; // Constant value, cannot be reassigned let userAge = 25; // Variable value, can be reassigned let userName = "John Doe"; // Block-scoped variable // Legacy JavaScript - Avoid using this var oldStyleVariable = "legacy"; // Function-scoped, causes issues // Examples of usage const MAX_ATTEMPTS = 3; // Constants in UPPER_CASE let currentAttempts = 0; // Variables in camelCase let isLoggedIn = false; // Boolean variables console.log("PI value:", PI); console.log("User age:", userAge); console.log("User name:", userName);

💡 Senior Engineer Tip:

Always use `const` by default, only use `let` when you need to reassign the variable. Never use `var` in modern JavaScript.

🔢 JavaScript Data Types

JavaScript has several built-in data types. Understanding them is crucial for writing effective code:

// 1. PRIMITIVE DATA TYPES // String - Text data const firstName = "Alice"; const lastName = 'Johnson'; // Single or double quotes const fullName = `${firstName} ${lastName}`; // Template literals (ES6+) // Number - Both integers and decimals const age = 30; const height = 5.8; const temperature = -15; const infinity = Infinity; const notANumber = NaN; // Boolean - True or false const isStudent = true; const hasGraduated = false; // Undefined - Variable declared but not assigned let undefinedVariable; console.log(undefinedVariable); // undefined // Null - Intentionally empty value const emptyValue = null; // Symbol (ES6+) - Unique identifier const uniqueId = Symbol('id'); // BigInt (ES2020+) - Large integers const bigNumber = 1234567890123456789012345678901234567890n; // 2. REFERENCE DATA TYPES // Object - Collection of key-value pairs const person = { name: "John", age: 30, city: "New York", isEmployed: true }; // Array - Ordered list of values const colors = ["red", "green", "blue"]; const numbers = [1, 2, 3, 4, 5]; const mixedArray = ["text", 42, true, null]; // Function - Reusable block of code function greetUser(name) { return `Hello, ${name}!`; } // Date - Represents dates and times const currentDate = new Date(); const specificDate = new Date('2025-01-01');

🔄 Type Conversion and Coercion

JavaScript can convert between data types automatically (coercion) or manually (conversion):

// AUTOMATIC TYPE COERCION (JavaScript does this automatically) console.log("5" + 3); // "53" (string concatenation) console.log("5" - 3); // 2 (numeric subtraction) console.log("5" * "2"); // 10 (both converted to numbers) console.log(true + 1); // 2 (true becomes 1) console.log(false + 1); // 1 (false becomes 0) // MANUAL TYPE CONVERSION (You control the conversion) // Converting to String const num = 42; const str1 = String(num); // "42" const str2 = num.toString(); // "42" const str3 = `${num}`; // "42" (template literal) // Converting to Number const text = "123"; const num1 = Number(text); // 123 const num2 = parseInt(text); // 123 (integer only) const num3 = parseFloat("123.45"); // 123.45 const num4 = +text; // 123 (unary plus operator) // Converting to Boolean const bool1 = Boolean("hello"); // true (non-empty string) const bool2 = Boolean(""); // false (empty string) const bool3 = Boolean(0); // false const bool4 = Boolean(1); // true const bool5 = !!text; // true (double negation) // CHECKING DATA TYPES console.log(typeof "hello"); // "string" console.log(typeof 42); // "number" console.log(typeof true); // "boolean" console.log(typeof undefined); // "undefined" console.log(typeof null); // "object" (JavaScript quirk!) console.log(typeof {}); // "object" console.log(typeof []); // "object" (arrays are objects!) // Better way to check arrays console.log(Array.isArray([])); // true console.log(Array.isArray({})); // false

⚠️ Common Gotchas:

  • typeof null returns "object" (this is a known JavaScript bug)
  • typeof [] returns "object" (use Array.isArray() to check for arrays)
  • Adding strings and numbers can lead to unexpected results
  • Empty strings, 0, null, undefined, and NaN are "falsy" values

🎯 Senior Engineer Best Practices

1. Variable Naming Conventions

// ✅ Good variable names (descriptive and clear) const userEmailAddress = "user@example.com"; const maxRetryAttempts = 3; const isUserLoggedIn = true; const calculateTotalPrice = () => { /* function code */ }; // ❌ Bad variable names (unclear and confusing) const e = "user@example.com"; // What is 'e'? const max = 3; // Max what? const flag = true; // What kind of flag? const calc = () => { /* code */ }; // Calculate what?

2. Defensive Programming

// ✅ Check types before using values function processUserAge(age) { // Type checking if (typeof age !== 'number' || isNaN(age)) { throw new Error('Age must be a valid number'); } // Range checking if (age < 0 || age> 150) { throw new Error('Age must be between 0 and 150'); } return age >= 18 ? 'adult' : 'minor'; } // ✅ Use default values function greetUser(name = 'Guest') { return `Hello, ${name}!`; } // ✅ Handle null/undefined safely const user = { profile: { name: 'John' } }; // Safe property access (optional chaining - ES2020) const userName = user?.profile?.name ?? 'Unknown';

3. Performance Considerations

// ✅ Declare variables in appropriate scope function processData() { const CHUNK_SIZE = 1000; // Constant at function level for (let i = 0; i < data.length; i++) { // Use 'let' for loop variables const item=data[i]; // Declare close to usage // Process item... } } // ✅ Use const for objects/arrays that won't be reassigned const userPreferences={ theme: 'dark' , language: 'en' }; // You can still modify the object userPreferences.theme='light' ; // This is fine! // ✅ Use meaningful constants const HTTP_STATUS={ OK: 200, NOT_FOUND: 404, SERVER_ERROR: 500 };
